DAY10

Flexbox

정의

웹 페이지에서 요소들을 정렬하고 배치할 때 사용하는 CSS 모델

가로 세로 정렬, 공간 분배, 반응형 디자인 등

필요한 이유

기존 레이아웃 단점

  • 세로 정렬이 매우 어려움
  • 아이템 간 간격 조절이 복잡함
  • 브라우저 크기 변화에 따라 자동 정렬이 어려움

이런 문제를 해결하기 위해 나온 것이 Flexbox

Flex Container에 적용되는 속성

1.display:flex

해당 요소를 Flex Container로 만들어 내부 자식 요소들을 flex item으로 배치
기본 축은 가로 방향이며 inline 요소는 display:inline-flex

2.flex-direction

주축의 방향 설정(기본값:row,오른쪽 방향)
  • row:기본,가로 방향으로 배치
  • row-reverse:오른쪽->왼쪽
  • column:세로 방향 위->아래
  • column-reverse:아래->위

3.justify-content

주축을 따라 item 정렬 방법 지정
flex-direction에 따라 가로/세로 방향이 바뀜
  • flex-start:시작점 정렬(왼쪽 or 위)
  • flex-end:끝점 정렬(오른쪽 or 아래)
  • center:가운데 정렬
  • space-between:양 끝 배치, 나머지는 동일 간격
  • space-around:아이템을 중심으로 동일한 간격
  • space-evenly:아이템과 공간 모두 동일 간격

4.flex-wrap

아이템이 한 줄에 다 들어가지 않을 때 줄바꿈 여부를 설정
  • nowrap:줄바꿈 안 함(기본값)
  • wrap:넘치면 다움 줄로 감
  • wrap-reverse:줄바꿈하되 아래 줄부터 위로 쌓음

5.align-items

교차축 기준으로 아이템 정렬
단일 줄의 정렬에 사용
  • flex-start:위쪽 정렬
  • flex-end:아래쪽 정렬
  • center:가운데 정렬
  • baseline:텍스트 기준선 기준으로 정렬
  • stretch:기본값. 높이나 너비가 자동으로 채워짐

6.align-content

여러 줄이 wrap 되었을 때 교차축에서 줄 간격을 정렬
align-items는 한 줄, align-content는 여러줄

Flex Item에 적용되는 속성

1.flex-grow

남은 공간을 얼마나 확장해서 차지할지 비율로 지정
0이면 남는 공간을 차지하지 않음
숫자가 클수록 더 많은 공간을 차지함

2.flex-shrink

공간이 부족할 때 얼마나 줄일 수 있는지 비율로 지정
0이면 줄어들지 않음
숫자가 클수록 더 많이 줄어듦

3.flex-basic

아이템의 기본 크기 설정(가로,세로 방향)
width,height와 비슷하지만 flex iem의 기준 크기로 사용됨

flex(속기형)

flex-grow, flex-shrink, flex-basis
           .item{
            flex:1 2 300px;
        }
        ->flex-grow: 1, flex-shrink: 2, flex-basis 300px

4.align-self

개별 아이템의 교차축 정렬 지정
align-items의 개별 버전
